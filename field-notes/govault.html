GoVault

A full Stratum V1 mining server written in Go — packaged as a native desktop app, a headless edge node, and an admin aggregator. Solo mine against your own node, or proxy through to any upstream pool. No fees. No operator.

Why Build This

Every home miner eventually asks the same question: why send my hashes to a pool operator I don't control, paying fees on rewards I can't verify? GoVault exists to answer that question. Point your Bitaxe, NerdAxe, or NerdMiner at GoVault and it handles the entire Stratum session — job generation, share validation, vardiff, and block submission — entirely on your own hardware.

Three Binaries, One Codebase

  • GoVault.exe — native desktop app (Wails v2 + Svelte 4). 1280×800 window, WebView2 shell, six UI themes. Run it on your mining rig's host machine or any Windows box on the same network.
  • edgenode.exe — headless server with a browser-based setup wizard. Designed for always-on deployment on a Pi, old laptop, or VPS. Registers with the admin panel so all your edge nodes aggregate into one view.
  • gostratum-admin.exe — polls registered edge nodes and consolidates their stats into a single dashboard.

Two Modes

Solo Mode

GoVault connects directly to a local full node via JSON-RPC getblocktemplate. The chain monitor polls every 10 seconds with a 500ms fast-detect on block changes — when a new block arrives, all connected miners get fresh work within half a second. If a miner finds a valid block, GoVault submits it directly to the network via submitblock. No middleman.

Proxy Mode

In proxy mode, GoVault maintains an outbound Stratum connection to an upstream pool (FirePool, any other pool, or a solo pool endpoint). Jobs relay from upstream to miners; shares validate locally and forward upstream when they meet the pool's difficulty. Each miner gets its own extranonce1 carved from a prefix of the upstream extranonce — correct nonce-space isolation without requiring the upstream to know about individual miners.

The edge node takes this further with per-miner pass-through: each connecting miner gets its own dedicated upstream TCP connection. From the pool's perspective, every miner is an independent worker with its own credentials and its own hashrate attribution.

VarDiff

Target shares per interval, retarget every N seconds, variance tolerance band before adjusting, idle halving on timeout. Worker difficulty persists to SQLite and restores on reconnect — a miner that was running at 512K difficulty doesn't have to ramp back up from scratch after a disconnect. Hardware auto-detection sets the starting difficulty per device type: NerdMiner starts much lower than a Bitaxe, which starts much lower than an Avalon Q.

One important detail: mining.suggest_difficulty is stored as a vardiff floor only — it does not override the current difficulty. Some ASIC firmware (BM1366) sends a hardware-filtered difficulty of 512 regardless of actual hashrate. Treating that as the target would cause severe hashrate underestimation.

The UI

Six themes ship in the desktop app: Nuclear, TRON, Vault-Tec, Crimson, Ultraviolet, and Plasma. The Vault-Tec theme carries the same design language as ZombieOS — amber gold on deep blue, scanline texture, corner bracket card accents. The edge node UI is a standalone HTML page with a rolling SVG hashrate chart (60-point window, gradient fill, current/peak/average labels) and a miners table with live share rates and accept/reject bars.

Supported Coins

BTC, BCH, DGB (SHA-256d — multi-algo aware network difficulty), BC2 (Bitcoin II), and XEC (eCash — mandatory miner fund and staking reward injected correctly into every coinbase). Address validation is per-coin, pool presets are included for FirePool, and node auto-detection probes the local RPC and reads cookie auth automatically.
