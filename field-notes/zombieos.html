The Problem With Stock

An Antminer S9 is a piece of 2017 hardware running firmware designed for a data centre — not a home lab. Stock Bitmain firmware gives you a basic web interface, no tuning control, and no insight into what's actually happening at the chip level. Braiins OS fixes the tuning side, but the UI is still generic. ZombieOS is the next layer: a complete firmware overlay that takes ownership of the entire stack.

The Architecture

ZombieOS runs on top of Braiins OS Legacy (OpenWrt-based). The overlay lives in /www/zombieos/ and consists of four HTML pages served by uhttpd, backed by two Lua CGI scripts:

  • zombieos-stats — reads Prometheus metrics from BOS+ (always gzip), UCI config, TOML config, and the autotune savefile. Returns a single JSON payload covering hashrate, temperature, fan state, chip health, and autotuner stage.
  • zombieos-ctl — handles all write operations: fan mode, frequency, voltage, profile selection, board toggles, pool config, restart.

The Prometheus layer in BOS+ 0.9.0 has its quirks — it reports hot_temperature 0 while the autotuner is active (a firmware bug). ZombieOS implements a three-tier fallback: Prometheus → UCI → direct TOML parse, so the displayed temperatures are always accurate regardless of autotuner state.

Per-Chip Voltage Autotuning

The autotuner runs four stages: RAMP, BASELINE, CHIP TUNE, and VERIFY. Stage 3 is where the real work happens — it sweeps (voltage, frequency) combinations for every chip individually, finding the minimum stable voltage for each one. With 4 weaker-than-average BM1387 chips on this board, Stage 3 took over four hours of continuous testing at ~211 seconds per test point.

After 29 iterations across multiple runs, Stage 4 VERIFY passed clean: 7.942 V, 0 slow chips, all 63 hashing. The profile is saved to NAND and survives every reboot.

One critical discovery: bosminer always divides the power target by 3 regardless of how many boards are installed. With a single board, you set the target at 3× your desired per-board budget. The ZombieOS dashboard shows the real board budget, not the misleading raw PSU slider value.

The UI

Four pages, each built from scratch with a Vault-Tec dual-colour theme — amber gold and deep blue, phosphor scanlines, corner bracket accents on every card. The dashboard loads with a CRT phosphor boot animation and staggered card entrances, then settles into a live-polling layout with:

  • Full-width live graphs — hashrate (2fr), temperature and power (1fr each)
  • Autotuner pipeline visualiser — 4-step progress indicator driven by real savefile stage data, not elapsed time
  • Chip health grid — 63 boxes, green/orange by per-chip threshold status
  • Block odds card — real BCH network difficulty, shows "1 in 2.3M odds" rather than an earnings estimate
  • WebGL globe — raw GLSL, 4 shader programs, Phong ocean shading, polar ice caps, atmosphere rim glow, live signal arc when shares are accepted

The Globe

The globe is written entirely in raw WebGL — no Three.js, no library of any kind. Four GLSL shader programs handle the sphere with ocean shading, continent fills, grid lines and signal arcs, atmospheric glow, and star points. The continents are 9 simplified closed polygons triangulated as TRIANGLE_FANs from their centroid. A 2D canvas overlay stacks above the WebGL canvas for text labels and ripple rings that fire on each accepted share.

Fan Watchdog

The fan control runs as a state machine inside the stats CGI: auto, floor, boost, and at_assist. The at_assist state takes over when the autotuner is active — BOS+ PID is bypassed and ZombieOS drives fan speed proportionally to board temperature via a two-segment curve. The watchdog re-evaluates every 5 seconds and adjusts both up and down with ±5% hysteresis to prevent oscillation.
